# Landing And Product Page Reference

Use this reference for landing pages, company sites, product pages, launch pages, and brand-led marketing surfaces.

## Direction

- Define the offer in one sentence before visual design: product, audience, outcome, and why now.
- Treat the first viewport as the brand/product thesis, not as a generic conversion block.
- Choose one primary artifact: product render, workflow object, customer environment, material detail, result state, or brand image. Build the hero around it.
- Set style intensity by category: utilitarian B2B pages usually need 5-7/10; premium product or launch pages can go 8-9/10; experimental editorial pages need explicit user or brand fit.
- Use one memorable spatial move: asymmetrical media crop, strong typographic axis, product-in-environment frame, scroll rhythm, proof rail, or grounded 3D/product moment.

## Required Structure

- First viewport: brand signal, clear offer, primary action, product/world signal, and a hint of the next section.
- Proof sequence: choose the minimum credible proof, not filler logos or fake metrics.
- Feature sequence: map features to user jobs, not generic icon cards.
- Product explanation: show actual workflow anatomy, before/after, state progression, or concrete output.
- Conversion area: make the next action obvious and honest. Do not invent pricing, claims, awards, or testimonials.

## Anti-Default Rules

- Do not add eyebrow/kicker/badge text above the H1 unless requested or present in accepted reference.
- Avoid centered hero plus bento grid unless the product truly benefits from that structure.
- When no voice is specified, write eloquent, professional, intelligent, high-class copy that is concrete to the offer, audience, and proof. If the user requests or implies a different vibe, match that voice exactly. Any phrasing is valid when it fits product truth, brand context, audience, or user-provided direction; make every claim earned.
- Avoid decorative dashboards when the product is not software.
- Avoid lifestyle imagery that could belong to any brand in the category.

## Concept Brief Additions

- Name the page's emotional job: trust, speed, luxury, clarity, craft, technical confidence, delight, urgency, or calm control.
- Name the hero artifact and why it proves the offer.
- Ask Image Gen for downstream rhythm, not just a hero: proof, feature/workflow, deeper product explanation, and final action.
- For multi-section pages, request section concepts that share gutters, type system, color rhythm, media treatment, and component geometry.

## Implementation Gates

- Preserve first viewport balance and next-section preview.
- Keep brand/product/place/object visible in the first viewport.
- Build sections from one token and component system; do not create a new card style per section.
- Verify desktop and mobile line breaks for H1, subcopy, nav, and CTAs.
- Remove any section that only fills visual rhythm without adding user decision value.
