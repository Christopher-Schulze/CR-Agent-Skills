# Style Archetypes Reference

Use this before Image Gen when visual direction is open or previous results feel samey. These are design worlds, not templates. Choose one primary archetype and at most one minor accent; do not blend several into a vague premium soup.

## Decision Rule

- Pick the archetype whose structure best serves the product job, not the one that looks most decorative.
- State the chosen archetype, the rejected obvious default, and the subject-tied signature move before briefing Image Gen.
- Use the archetype to decide spacing, type, component behavior, asset role, and motion. Do not copy sample layouts mechanically.

## Editorial Utility

- Use when: content-heavy products, documentation-like surfaces, professional services, analysis tools, portfolios, and calm product explanations.
- Visual thesis: sharp reading rhythm, strong typographic axis, restrained proof objects, and purposeful media interruptions.
- Layout/spacing: open columns, disciplined gutters, generous section starts, few cards, visible next-section rhythm.
- Typography: expressive display role with precise body and caption roles; avoid all-text-same-weight layouts.
- Component behavior: quiet controls, crisp links, subtle dividers, compact proof rails, clear focus states.
- Avoid: newspaper cosplay, decorative hairlines everywhere, beige default editorial, and walls of text without decision structure.

## Dense Mission Control

- Use when: dashboards, operations, logistics, monitoring, CRM, admin, planning, analytics, and triage surfaces.
- Visual thesis: information hierarchy first; selected/detail model, status semantics, and scan speed create the premium feeling.
- Layout/spacing: dense tables, queues, timelines, maps, inspectors, and command bars with calm margins and strong grouping.
- Typography: compact UI/control text, tabular data rhythm, explicit label/value hierarchy.
- Component behavior: row states, filters, sort, selected panels, status tokens, keyboardable controls, visible focus.
- Avoid: marketing heroes, KPI-wall filler, bento cards without workflow, fluffy prose, and empty glass chrome.

## Atelier OS

- Use when: creative tools, editors, builders, media/canvas products, AI generation workbenches, and document/design surfaces.
- Visual thesis: the work product owns attention; chrome feels crafted, dense, and quiet.
- Layout/spacing: central canvas/document/media, side inspector, toolbar rails, status bar, compact layers or assets.
- Typography: precise UI text, small labels, clear mode names, readable work product typography.
- Component behavior: selected object states, handles, tool modes, inspector fields, undo/redo/export paths.
- Avoid: chrome louder than canvas, fake creative widgets, oversized cards, and generic SaaS dashboard framing.

## Luxury Transaction

- Use when: premium checkout, booking, hospitality, finance, fashion, wellness, venues, and high-trust purchase flows.
- Visual thesis: trust, material quality, and recoverability; the transaction feels calm and expensive without slowing action.
- Layout/spacing: one decision per step, tactile summary object, receipt/itinerary/product preview, stable CTA region.
- Typography: elegant but highly legible; labels and errors are plain and concrete.
- Component behavior: strong input craft, validation, confirmation, progress, disabled/loading, and reachable mobile actions.
- Avoid: spectacle over trust, hidden fees/states, too much copy, decorative badges, and low-contrast luxury styling.

## Cinematic Product World

- Use when: consumer brands, hardware, venues, food, fashion, travel, entertainment, and object-led launches.
- Visual thesis: one dominant object/environment/ritual carries the first viewport; copy and UI stay disciplined.
- Layout/spacing: strong crop, controlled negative space, fewer components, image-led transitions, visible next section.
- Typography: strong display identity balanced by restrained support copy and clear CTA/control text.
- Component behavior: minimal but refined controls, image-safe focus rings, considered hover/active treatment.
- Avoid: dark blurred stock mood, generic gradients, fake app panels, too many small images, and copy overloading the hero.

## Swiss Industrial SaaS

- Use when: B2B software, developer tools, infrastructure, finance ops, security, and serious professional products.
- Visual thesis: clarity, restraint, grid discipline, and precise component/state semantics.
- Layout/spacing: strong columns, rails, tables, specs, diagrams, product previews, and compact proof modules.
- Typography: neutral but intentional; data/control type must not feel browser-default.
- Component behavior: exact buttons/tabs/forms/tables, visible focus, meaningful disabled/loading/error states.
- Avoid: cold template sameness, fake technical diagrams, decorative terminal/code blocks, and all-gray monotony.

## Tactile Mobile Commerce

- Use when: mobile shopping, booking, ordering, scheduling, onboarding, wallet, and consumer transaction flows.
- Visual thesis: thumb-first confidence with tactile selectors, previews, progress, and immediate confirmation.
- Layout/spacing: one primary decision per screen, stable bottom action, compact summary object, generous touch targets.
- Typography: short labels, high-readability prices/statuses, concise helper/error copy.
- Component behavior: selected chips, steppers, calendars, cards, maps, receipts, loading and success states.
- Avoid: desktop density squeezed into mobile, long explanations above CTA, tiny controls, and decorative empty space.

## Playfield-First Game

- Use when: browser games, simulations, playful tools, interactive scenes, and toy apps.
- Visual thesis: gameplay readability comes before UI ornament; player, hazards, goals, rewards, route, and HUD are instantly legible.
- Layout/spacing: full-bleed or dominant playfield, stable HUD, safe control zones, parallax only when it preserves readability.
- Typography: short HUD labels, readable score/status, no tutorial copy blocking action.
- Component behavior: input feedback, restart, pause, win/lose, score/progress, collision-aligned art.
- Avoid: decorative scene detail crowding the route, CSS placeholder art when production assets are required, and UI chrome that competes with play.
